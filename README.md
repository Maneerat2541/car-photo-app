# ระบบเก็บข้อมูลรูปรถ (Car Photo App)

แอปถ่ายรูปรถ 3 ขั้น (บาร์รถ · หลังรถ · ทะเบียนรถ) บังคับถ่ายจากกล้อง
ล็อกวันที่-เวลา-พิกัด GPS ลงบนรูป แล้วส่งเข้าคลาวด์แยกตามวันที่ พร้อมหน้าจอ
ออฟฟิศแบบเรียลไทม์ ปรับปรุงจากดีไซน์ต้นฉบับ (Claude Design) ให้เป็นแอปจริง
ที่รันได้ทันทีโดยไม่ต้อง build

## ไฟล์

| ไฟล์ | หน้าที่ |
|------|--------|
| `index.html`   | แอปมือถือสำหรับเจ้าหน้าที่ภาคสนาม — ถ่าย 3 ขั้น + คลังรูป |
| `office.html`  | หน้าจอออฟฟิศ (เรียลไทม์) — ฟีดรูป, กราฟแนวโน้ม, ดาวน์โหลด ZIP, ลบรายวัน |
| `sync.js`      | เลเยอร์ซิงก์ข้อมูล (local/BroadcastChannel หรือ Firebase) |
| `image-slot.js`| คอมโพเนนต์ช่องรูปตัวอย่าง (แตะ/วางรูปเพื่อใส่ตัวอย่างแต่ละขั้น) |

## วิธีรัน

ต้องเสิร์ฟผ่าน HTTP (ใช้ ES module `import`) — เปิดไฟล์ตรง ๆ แบบ `file://`
จะไม่ทำงาน และกล้องต้องใช้ HTTPS หรือ `localhost`

```bash
# ตัวอย่าง (เลือกอย่างใดอย่างหนึ่ง)
npx serve .
python -m http.server 5177
```

แล้วเปิด `http://localhost:5177/` (มือถือ) และ `http://localhost:5177/office.html` (ออฟฟิศ)

> ในเครื่องนี้ไม่มี Node/Python — จึงมี static server สำรองที่ `.claude/serve.ps1`
> (สั่งรันผ่าน preview ชื่อ `carphoto` ใน `.claude/launch.json`)

## โหมดข้อมูล

- **ค่าเริ่มต้น (local):** เก็บใน `localStorage` และซิงก์ข้ามแท็บบนเครื่องเดียวกัน
  ผ่าน `BroadcastChannel` — เหมาะกับเดโม/ทดลอง
- **ออนไลน์เรียลไทม์ข้ามอุปกรณ์ (Firebase):** แก้ที่ไฟล์ **`config.js`** ไฟล์เดียว
  (ใช้ได้ทั้ง `index.html` และ `office.html`) — ดูขั้นตอนเต็มใน **[DEPLOY.md](DEPLOY.md)**

  แต่ละรูปถูกเขียนเป็นเอกสารแยกใน `<collection>_shots` เพื่อเลี่ยงลิมิต 1MiB/doc
  และมี `<collection>_stats` เก็บยอดรายวันไว้ให้กราฟแนวโน้มถูกต้องแม้ลบรูปเก่าออกแล้ว
  (พอดีกับแพ็กฟรี Firebase Spark)

การขึ้นระบบจริง (GitHub Pages + Firebase + Google Sheet) ดูรายละเอียดใน **[DEPLOY.md](DEPLOY.md)**

## เกี่ยวกับที่มา

โฟลเดอร์ `handoff_extracted/` และไฟล์ `.zip` คือดีไซน์ต้นฉบับจาก Claude Design
เก็บไว้อ้างอิงเท่านั้น — โค้ดที่ใช้งานจริงคือไฟล์ที่ราก (root) ของโปรเจกต์
